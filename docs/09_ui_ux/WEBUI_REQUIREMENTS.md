# Web UI 要件定義

ServiceMatrix Web User Interface Requirements

Version: 1.0
Status: Active
Classification: Internal Design Document
Last Updated: 2026-03-02

---

## 1. 概要

本ドキュメントは、ServiceMatrix の Web ユーザーインターフェースに関する
機能要件・非機能要件・技術要件を定義する。

ServiceMatrix は GitHub ネイティブ基盤であるが、
運用オペレーター・マネージャー・監査担当者が直感的に操作できる
Web UI を提供することで、統治の可視性と操作性を向上させる。

---

## 2. 対象ユーザーと利用シナリオ

### 2.1 対象ユーザー

| ロール | 主な利用目的 | 利用頻度 |
|--------|-------------|----------|
| Admin | システム設定・ユーザー管理・統治ポリシー管理 | 日次 |
| Change Manager | 変更要求の承認・CAB運営・変更スケジュール管理 | 日次 |
| Incident Manager | インシデント対応統括・エスカレーション判断 | 常時 |
| Operator | インシデント登録・サービス要求処理・日常運用 | 常時 |
| Viewer | レポート閲覧・SLA状況確認 | 週次 |
| AI Agent | API経由でのデータ参照・更新（UI直接操作なし） | 常時 |

### 2.2 主要利用シナリオ

| # | シナリオ | ロール | 画面 |
|---|---------|--------|------|
| S-01 | P1インシデント発生時の即時登録と対応開始 | Operator | インシデント登録画面 |
| S-02 | SLA違反アラートの確認と対応指示 | Incident Manager | ダッシュボード・SLA画面 |
| S-03 | 変更要求の作成と影響分析の確認 | Operator | 変更申請画面 |
| S-04 | CAB会議での変更承認判断 | Change Manager | 変更承認画面 |
| S-05 | 月次SLAレポートの確認 | Viewer | SLAレポート画面 |
| S-06 | CMDB構成アイテムの検索と関連確認 | Operator | CMDB画面 |
| S-07 | 監査証跡の検索と証拠エクスポート | Admin | 監査ログ画面 |
| S-08 | AI Agent活動ログの確認と逸脱検知 | Admin | AI活動ログ画面 |

## 5. 非機能要件

### 5.1 パフォーマンス要件

| 項目 | 要件 | 目標値 |
|------|------|--------|
| 初期ロード時間 | ダッシュボードの初期表示 | 3秒以内 |
| ページ遷移 | SPA内画面遷移 | 500ms以内 |
| 検索応答 | インシデント・CI検索 | 2秒以内 |
| リアルタイム更新 | 通知・ステータス変更の反映 | 5秒以内 |
| 同時接続 | 同時利用ユーザー数 | 50ユーザー以上 |

### 5.2 ブラウザ対応

| ブラウザ | バージョン | サポートレベル |
|---------|-----------|--------------|
| Google Chrome | 最新2バージョン | 完全サポート |
| Microsoft Edge | 最新2バージョン | 完全サポート |
| Mozilla Firefox | 最新2バージョン | 完全サポート |
| Safari | 最新2バージョン | 基本サポート |

### 5.3 レスポンシブ対応

| デバイス | 画面幅 | サポートレベル |
|---------|--------|--------------|
| デスクトップ | 1280px以上 | 完全サポート（主要ターゲット） |
| タブレット | 768px〜1279px | 基本サポート（閲覧中心） |
| モバイル | 768px未満 | 通知確認・緊急操作のみ |

### 5.4 アクセシビリティ

- WCAG 2.1 レベル AA 準拠を目標とする
- キーボードナビゲーション対応
- スクリーンリーダー対応（ARIA属性の適切な付与）
- カラーコントラスト比 4.5:1 以上

### 5.5 セキュリティ要件

| 項目 | 要件 |
|------|------|
| 認証 | GitHub OAuth 2.0 による認証 |
| セッション管理 | セッションタイムアウト：30分（設定可能） |
| CSRF対策 | トークンベースのCSRF防御 |
| XSS対策 | 出力エスケープ・Content Security Policy |
| 通信暗号化 | HTTPS（TLS 1.3以上）必須 |
| 権限制御 | ロールベースアクセス制御（RBAC） |

## 6. 技術スタック

### 6.1 フロントエンド

| カテゴリ | 技術 | 選定理由 |
|---------|------|---------|
| フレームワーク | React 19 / Next.js 15 | コンポーネントベース、SSR対応、エコシステム |
| 状態管理 | Zustand | 軽量、TypeScript親和性 |
| UIコンポーネント | shadcn/ui | カスタマイズ性、アクセシビリティ |
| スタイリング | Tailwind CSS | ユーティリティファースト、一貫性 |
| グラフ・チャート | Recharts | React親和性、カスタマイズ性 |
| フォーム | React Hook Form + Zod | バリデーション、型安全性 |
| テーブル | TanStack Table | 高機能テーブル、ソート・フィルタ・ページネーション |
| リアルタイム通信 | WebSocket / Server-Sent Events | リアルタイム更新 |

### 6.2 バックエンド（API）

| カテゴリ | 技術 | 選定理由 |
|---------|------|---------|
| ランタイム | Node.js / Bun | 高速、TypeScript ネイティブ |
| API フレームワーク | Hono | 軽量、高速、TypeScript対応 |
| 認証 | GitHub OAuth 2.0 | GitHub ネイティブ連携 |
| データアクセス | Drizzle ORM | 型安全、軽量 |

## 7. リアルタイム更新要件

### 7.1 更新方式

| イベント種別 | 更新方式 | 間隔 |
|------------|---------|------|
| インシデントステータス変更 | WebSocket | 即時 |
| SLAカウンター | ブラウザローカル計算 | 1秒ごと |
| 通知 | WebSocket | 即時 |
| ダッシュボードKPI | SSE（Server-Sent Events） | 30秒ごと |
| CMDB更新 | ポーリング | 60秒ごと |
| 監査ログ | ポーリング | 120秒ごと |

### 7.2 WebSocket接続管理

- 接続断時の自動再接続（指数バックオフ: 1s, 2s, 4s, 8s, 最大30s）
- ハートビート間隔: 30秒
- 接続状態インジケーターの表示
- オフライン時のキューイングとオンライン復帰時の一括同期

## 9. データ表示要件

### 9.1 一覧画面共通要件

- ページネーション（デフォルト: 25件/ページ）
- カラムソート（昇順・降順）
- フィルター（ステータス、優先度、担当者、日付範囲）
- クイックフィルター（よく使うフィルターのプリセット）
- 一括選択・一括操作
- CSV エクスポート

### 9.2 詳細画面共通要件

- タブ構成（基本情報、タイムライン、関連アイテム、変更履歴）
- インライン編集（権限に応じて）
- コメント・ノート機能
- 添付ファイル表示
- 関連Issue/PRへのリンク
