Version: 1.0
Status: Active

---

## 1. 文書の目的

本文書は、ServiceMatrix プロジェクトの戦略目標を体系的に定義する。
VISION_AND_MISSION.md で示された方向性を、
測定可能かつ達成可能な目標に分解したものである。

各戦略目標は、BSC（Balanced Scorecard）の4つの視点を参考に構造化し、
KGI（Key Goal Indicator）と KPI（Key Performance Indicator）を設定する。

---

## 2. 戦略目標の体系

ServiceMatrix の戦略目標は、以下の5つの戦略テーマに分類される。

| # | 戦略テーマ | 概要 |
|---|-----------|------|
| SO-1 | サービス統治の確立 | 4軸マトリクスモデルによる統治基盤の構築 |
| SO-2 | 運用効率の最大化 | AI 駆動の自動化による運用負荷の削減 |
| SO-3 | コンプライアンスの内在化 | 監査対応を設計に組み込む |
| SO-4 | 継続的改善の制度化 | データ駆動の改善サイクルの確立 |
| SO-5 | 組織能力の向上 | 統治文化の醸成と人材育成 |

---

## 3. SO-1: サービス統治の確立

### 3.1 目標概要

Process × Intelligence × DevOps × Governance の4軸マトリクスモデルに基づく
IT サービス統治基盤を構築し、すべてのサービス運用活動を
構造化された統治フレームワーク内で実施する。

### 3.2 KGI（重要目標達成指標）

| # | KGI | 目標値 | 達成期限 |
|---|-----|--------|----------|
| KGI-1.1 | 統治カバレッジ（統治対象プロセスの実装率） | 100% | 2027 Q2 |
| KGI-1.2 | 統治成熟度レベル（CMMI ベース） | レベル3 以上 | 2027 Q4 |
| KGI-1.3 | ポリシー技術的強制率 | 90%以上 | 2027 Q2 |

### 3.3 KPI（重要業績評価指標）

| # | KPI | 測定方法 | 目標値 | 測定頻度 |
|---|-----|----------|--------|----------|
| KPI-1.1 | プロセス定義完了率 | 定義済プロセス / 全対象プロセス | 100% | 月次 |
| KPI-1.2 | 状態遷移モデル実装率 | 実装済遷移 / 設計済遷移 | 100% | 月次 |
| KPI-1.3 | 承認フロー稼働率 | 承認フロー経由変更 / 全変更 | 95%以上 | 週次 |
| KPI-1.4 | GitHub Actions 統治チェック網羅率 | 自動チェック項目 / 全チェック項目 | 80%以上 | 月次 |

### 3.4 主要施策

1. 全コアプロセスの状態遷移モデルを設計・実装する
2. 承認制御モデルを GitHub PR フローに統合する
3. GitHub Actions による統治ポリシーの自動チェックを実装する
4. 統治ダッシュボードによるリアルタイム可視化を実現する
5. 四半期ごとの統治成熟度アセスメントを実施する

## 8. 戦略目標間の関係性

各戦略目標は独立ではなく、相互に関連し支え合う構造を持つ。

```
SO-5（組織能力）
    ↓ 支える
SO-1（統治確立） ← 基盤 → SO-3（コンプライアンス）
    ↓ 実現する              ↓ 保証する
SO-2（運用効率） ←─────→ SO-4（継続改善）
    ↑ フィードバック
```

**関係性の説明**

- SO-1（統治確立）が全体の基盤であり、他の目標の前提条件となる
- SO-2（運用効率）は SO-1 の統治基盤上で AI 自動化を実現する
- SO-3（コンプライアンス）は SO-1 の統治品質を保証する
- SO-4（継続改善）は SO-2 の運用データを入力として改善サイクルを回す
- SO-5（組織能力）はすべての目標を人的側面から支える
