# コンプライアンスチェックリスト

ServiceMatrix Compliance Checklist

Version: 1.0
Status: Active
Classification: Internal Audit Document
Last Updated: 2026-03-02

---

## 1. 概要

本ドキュメントは、ServiceMatrix が準拠すべきコンプライアンス要件の
チェックリストを提供する。
J-SOX / ISO 20000 / ISO 27001 の各要件に対する
実装状況・エビデンス有無・対応状態を一覧化する。

---

## 2. チェックリスト運用

### 2.1 運用ルール

| 項目 | 内容 |
|------|------|
| レビュー頻度 | 四半期（3月、6月、9月、12月） |
| 責任者 | Auditor（コンプライアンス担当） |
| 承認者 | Admin（最終承認） |
| 記録方法 | 本ドキュメントの更新 + GitHub Issue で記録 |
| 状態表記 | 適合 / 部分適合 / 不適合 / 対象外 / 未評価 |

### 2.2 状態定義

| 状態 | 説明 | 対応 |
|------|------|------|
| 適合 | 要件を完全に満たしている | 維持 |
| 部分適合 | 要件を部分的に満たしている | 改善計画策定 |
| 不適合 | 要件を満たしていない | 即時改善必要 |
| 対象外 | 当該システムには該当しない | 根拠を記録 |
| 未評価 | まだ評価を実施していない | 次回レビューで評価 |

## 8. 定期レビュープロセス

### 8.1 レビュースケジュール

| レビュー | 頻度 | 担当 | 内容 |
|---------|------|------|------|
| チェックリスト更新 | 四半期 | Auditor | 状態の更新、新規要件の追加 |
| ギャップ分析 | 半年 | Auditor + Admin | 不適合項目の改善計画策定 |
| 外部監査準備 | 年次 | Auditor | 監査対応準備、パッケージ作成 |
| 法規制変更対応 | 随時 | Auditor | 新規法規制要件の反映 |

### 8.2 更新プロセス

```
変更トリガー発生
  |
  v
影響分析
  |
  v
チェックリスト更新案作成
  |
  v
Auditor レビュー
  |
  v
Admin 承認
  |
  v
チェックリスト更新（Change Issue → PR → 承認）
  |
  v
関連ドキュメント更新
```
