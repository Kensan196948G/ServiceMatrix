# 監査証跡マッピング

ServiceMatrix Audit Evidence Mapping

Version: 1.0
Status: Active
Classification: Confidential - Audit Document
Last Updated: 2026-03-02

---

## 1. 概要

本ドキュメントは、ServiceMatrix プロジェクトにおけるJ-SOX（金融商品取引法に基づくIT全般統制）
および ISO/IEC 20000-1 準拠のための監査証跡マッピングを定義する。
各統制要件と、それを裏付ける証跡・証跡収集箇所・収集方法を対応づける。

---

## 2. 適用規制・標準

| 規制・標準 | 適用範囲 | 主要統制カテゴリ |
|----------|---------|---------------|
| J-SOX（金融商品取引法24条の4の4） | IT全般統制 | アクセス管理、変更管理、運用管理、ITインフラ |
| ISO/IEC 20000-1:2018 | サービス管理システム | インシデント、変更、問題、リリース、可用性 |
| ITIL 4 | プロセス基準 | 各ITSMプロセス |

## 4. ISO/IEC 20000-1 プロセス証跡マッピング

| プロセス | 要件 | 証跡 | 保存場所 |
|---------|------|------|---------|
| インシデント管理 | インシデントの記録と追跡 | incidents テーブル全レコード | DB |
| 問題管理 | 問題の記録・根本原因分析 | problems テーブル + RCA記録 | DB + GitHub |
| 変更管理 | 変更の記録・承認・実施 | change_requests + approvals テーブル | DB |
| リリース管理 | リリースの計画・実施記録 | GitHub Releases + deploy ログ | GitHub + DB |
| サービスレベル管理 | SLA遵守状況の記録 | sla_measurements テーブル | DB |
| 継続的改善 | 改善活動の記録 | GitHub Issue + 定期レビュー記録 | GitHub |

## 6. 証跡保存ポリシー

### 6.1 保存期間

| 証跡種別 | 保存期間 | 根拠 |
|---------|---------|------|
| J-SOX関連証跡 | 7年 | 金融商品取引法要件 |
| ISO 20000 プロセス記録 | 3年 | ISO/IEC 20000-1 要件 |
| セキュリティ監査ログ | 5年 | セキュリティポリシー |
| AI判断ログ | 5年 | AI統治ポリシー |
| テスト結果 | リリース後2年 | 内部規程 |
| インフラ設定履歴 | 5年 | 変更管理要件 |

### 6.2 証跡の完全性保護

| 保護手段 | 説明 |
|---------|------|
| 監査ログの書き込み専用 | 作成のみ可能、変更・削除不可 |
| タイムスタンプの改竄防止 | UTC タイムスタンプ + シーケンス番号 |
| ハッシュチェーン | 各ログエントリに前レコードのハッシュを付与 |
| 定期的なハッシュ検証 | 日次で証跡完全性の自動検証 |
| バックアップ | 証跡データは別リージョンにレプリカ保持 |
