# 監査エビデンスマッピング

ServiceMatrix Audit Evidence Mapping

Version: 1.0
Status: Active
Classification: Internal Audit Document
Last Updated: 2026-03-02

---

## 1. 概要

本ドキュメントは、ServiceMatrix における監査エビデンスの
マッピング定義を提供する。
J-SOX / ISO 20000 / ISO 27001 の各統制要件に対し、
システムが自動収集するエビデンスの対応関係を体系的に整理する。

---

## 2. エビデンスマッピング原則

### 2.1 基本方針

| 原則 | 説明 |
|------|------|
| 自動収集優先 | エビデンスは可能な限りシステムが自動収集する |
| 改竄防止 | 収集されたエビデンスは変更不可（Immutable）とする |
| 完全性保証 | エビデンスのハッシュチェーンにより完全性を保証する |
| 即時検索 | 監査要求時に即座にエビデンスを検索・提示可能とする |
| 保持期間遵守 | 法的要件に基づく保持期間を遵守する |
| トレーサビリティ | すべてのエビデンスが統制要件に紐づくこと |

### 2.2 エビデンス種別

| 種別 | 説明 | 収集方法 |
|------|------|---------|
| システムログ | 監査トレイル、アクセスログ、操作ログ | 自動（リアルタイム） |
| 変更記録 | RFC、承認記録、実施記録 | 自動（GitHub Issue/PR） |
| 設定記録 | CMDB構成情報、設定変更履歴 | 自動（バージョン管理） |
| テスト結果 | テストレポート、カバレッジ | 自動（CI/CD） |
| レビュー記録 | PRレビューコメント、承認 | 自動（GitHub PR） |
| インシデント記録 | インシデント対応記録、根本原因分析 | 自動 + 手動 |
| SLA記録 | SLA達成率、違反記録 | 自動（SLA Engine） |
| AI判断記録 | AI意思決定ログ | 自動（Agent Coordinator） |

## 6. エビデンスとシステムコンポーネントの対応

| コンポーネント | 生成するエビデンス | 統制カバレッジ |

| Incident Manager | インシデント記録、対応時間、解決記録 | ITGC-09, 8.1, A.16 |
| CMDB Core | CI一覧、変更履歴、依存関係 | 9.3, A.8 |
| SLA Engine | SLA達成率、違反記録、計測ログ | 6.1 |
| Audit Trail Manager | 全操作の監査ログ | A.12.4, ITAC-02 |
| CI/CD Pipeline | テスト結果、セキュリティスキャン結果 | ITGC-05, A.14.2 |
| Agent Coordinator | AI判断ログ、入出力記録 | AI統治要件 |
| PR Governance | PR作成/レビュー/承認記録、SoD検証結果 | ITGC-06, ITAC-04 |

### 6.2 エビデンス保存先

| エビデンス種別 | 一次保存先 | 二次保存先 | 保持期間 |
|--------------|-----------|-----------|---------|
| 監査ログ | PostgreSQL（追記専用テーブル） | オブジェクトストレージ | 7年 |
| 変更記録 | GitHub Issues/PRs | バックアップストレージ | 7年 |
| テスト結果 | CI/CDアーティファクト | オブジェクトストレージ | 3年 |
| 設定記録 | Git リポジトリ | バックアップストレージ | 永続 |
| SLA記録 | PostgreSQL | オブジェクトストレージ | 7年 |
| AI判断記録 | PostgreSQL（追記専用） | オブジェクトストレージ | 7年 |
| インシデント記録 | GitHub Issues | バックアップストレージ | 7年 |

## 7. エビデンス検索・提示

### 7.1 検索インタフェース

| 検索方法 | 対象 | 用途 |
|---------|------|------|
| 統制IDによる検索 | 統制要件に紐づくエビデンス一覧 | 監査対応 |
| 期間指定検索 | 指定期間内のエビデンス | 定期監査 |
| コンポーネント別検索 | 特定コンポーネントのエビデンス | 個別確認 |
| エビデンス種別検索 | 特定種別のエビデンス一覧 | カテゴリ別確認 |
| フリーテキスト検索 | 全文検索 | 調査・分析 |

### 7.2 エビデンスパッケージ

監査対応時に、必要なエビデンスを一括でパッケージ化して提供する。

| パッケージ | 内容 | 対象監査 |
|-----------|------|---------|
| J-SOXパッケージ | ITGC/ITAC全統制のエビデンス | J-SOX監査 |
| ISO 20000パッケージ | サービス管理全条項のエビデンス | ISO 20000審査 |
| ISO 27001パッケージ | 情報セキュリティ統制のエビデンス | ISO 27001審査 |
| カスタムパッケージ | 指定統制IDのエビデンス | 個別監査要求 |

### 7.3 エビデンスレポート形式

| 形式 | 用途 | 内容 |
|------|------|------|
| PDF | 監査提出用 | エビデンス本文 + メタデータ + ハッシュ |
| CSV | 分析用 | エビデンス一覧 + メタデータ |
| JSON | システム連携用 | 構造化エビデンスデータ |
| ZIP | 一括提出用 | 複数エビデンスのアーカイブ |

## 8. エビデンス完全性の保証

### 8.1 ハッシュチェーン

| 項目 | 仕様 |
|------|------|
| アルゴリズム | SHA-256 |
| チェーン構造 | 各エビデンスのハッシュに前エビデンスのハッシュを含む |
| 検証頻度 | 日次（自動） |
| 検証失敗時 | Critical アラート + インシデント自動起票 |

### 8.2 改竄検知

| 検知手法 | 対象 | 頻度 |
|---------|------|------|
| ハッシュチェーン検証 | 監査ログ | 日次 |
| チェックサム検証 | エビデンスファイル | 日次 |
| アクセスログ監視 | エビデンス保存先 | リアルタイム |
| 整合性チェック | DB とオブジェクトストレージ | 週次 |
